# Лабораторная работа №7
## Оценка покрытия кода тестами и доведение покрытия ветвей до 80%

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 06.04.2026

## 1. Цель работы

Подключить инструмент оценки покрытия кода тестами, проанализировать unit-тесты из ЛР4 и дополнить их до достижения покрытия не менее 80% ветвей.

## 2. Использованный инструмент

Для проекта на Gradle подключен `JaCoCo`.

В конфигурации сборки добавлены:

- генерация `HTML` и `XML` отчетов;
- автоматический запуск отчета после тестов;
- проверка минимального уровня `branch coverage`;
- область проверки ограничена классами бизнес-логики из ЛР4.

## 3. Дописанные unit-тесты

Для повышения покрытия ветвей добавлены тесты на сценарии:

- слишком длинное имя компонента;
- пустой идентификатор соединения;
- пустое имя сигнала;
- обрезку пробелов у идентификатора соединения и имени сигнала;
- отсутствие совпадений по типу компонента;
- отсутствие изолированных компонентов при полной связности.

## 4. Проверка покрытия

Покрытие проверяется Gradle-задачей:

```bash
./gradlew jacocoTestCoverageVerification
```

Требование лабораторной:

- покрытие ветвей для классов ЛР4 не менее `80%`.

## 5. Результат

В рамках лабораторной работы №7 подключен `JaCoCo`, unit-тесты из ЛР4 дополнены, а проверка покрытия ветвей встроена в сборку проекта.
