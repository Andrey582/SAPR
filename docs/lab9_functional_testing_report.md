# Лабораторная работа №9
## Автоматизированное функциональное тестирование API

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 06.04.2026

## 1. Цель работы

Настроить и выполнить простой сценарий автоматизированного функционального тестирования API для симуляции полного пользовательского сценария работы системы.

## 2. Реализованный API-сценарий

Для лабораторной добавлен минимальный REST API с in-memory хранением:

- `POST /api/workspace/reset` — очистка рабочего пространства;
- `POST /api/components` — создание компонента схемы;
- `POST /api/connections` — создание соединения между компонентами;
- `GET /api/summary` — получение итогового состояния схемы.

Сценарий проверки:

1. Очистить рабочее пространство.
2. Создать компонент `R1`.
3. Создать компонент `C1`.
4. Создать соединение `W1` между `R1` и `C1`.
5. Проверить итоговую сводку схемы.

## 3. Инструмент автоматизации

Подготовлена коллекция Postman:

- `postman/lab9_functional_scenario.postman_collection.json`

Так как в текущем окружении отсутствует `Postman/Newman`, для фактического запуска сценария дополнительно подготовлен shell-скрипт:

- `scripts/lab9_functional_api_scenario.sh`

Скрипт выполняет те же шаги, что и коллекция Postman, и проверяет ответы API на каждом этапе.

## 4. Ожидаемый результат сценария

После завершения сценария система должна вернуть:

- `componentCount = 2`
- `connectionCount = 1`
- `isolatedComponentIds = []`
- `invalidConnectionIds = []`

В рамках лабораторной работы №9 настроен и выполнен автоматизированный функциональный сценарий работы API, а также подготовлена Postman-коллекция для воспроизведения этого теста.
