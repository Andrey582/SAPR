# Лабораторная работа №1
## Реализация доски проекта в GitHub Actions

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Лабораторная работа:** №1  
**Инструмент:** GitHub Actions  
**Проект:** `CAD-Schemes`

## 1. Цель работы

Реализовать аналог проектной доски для задач САПР схем в экосистеме GitHub с автоматической сборкой и публикацией состояния доски.

## 2. Реализация

Для лабораторной использована следующая схема:

- исходные данные доски хранятся в файле `board/lab1_github_board.csv`;
- workflow `.github/workflows/lab1-github-board.yml` запускается вручную или при изменении доски;
- скрипт `scripts/generate_lab1_github_board.sh` формирует markdown-снимок текущего состояния;
- GitHub Actions публикует snapshot в `GITHUB_STEP_SUMMARY` и загружает artifact `lab1-github-board`.

## 3. Структура колонок

Доска содержит колонки:

- `Backlog`
- `To Do`
- `In Progress`
- `Code Review`
- `Testing`
- `Done`

## 4. Бэклог и Sprint 1

В доске отражены:

- backlog проекта по теме САПР схем;
- задачи Sprint 1;
- исполнитель;
- приоритет;
- story points;
- сроки выполнения.

## 5. Результат запуска workflow

После запуска workflow формируются:

- markdown-снимок доски;
- исходный CSV с задачами;
- данный отчет в составе artifact.

Artifact:

- `lab1-github-board`

## 6. Вывод

В рамках лабораторной работы №1 доска задач реализована в GitHub Actions: состояние бэклога и спринта хранится в репозитории, автоматически преобразуется в snapshot и публикуется как результат выполнения workflow.
