# Лабораторная работа №11
## Изменение требований, рефакторинг и регрессионное тестирование

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 06.04.2026

## 1. Цель работы

Сымитировать доработку одной из функций приложения, выполнить рефакторинг кода и подтвердить с помощью полного regression suite, что остальные функции не были сломаны.

## 2. Изменение требований

Изменены требования к функции:

- `CircuitAnalysisService.findInvalidConnectionIds(...)`

Новое требование:

- соединение считается невалидным, если у него пустое имя сигнала, даже если оба компонента существуют и соединение не является self-loop.

Старые требования сохранены:

- отсутствие source-компонента;
- отсутствие target-компонента;
- self-loop соединение.

## 3. Рефакторинг

Функция рефакторена для упрощения логики и повышения читаемости:

- вынесена общая проверка `isInvalidConnection(...)`;
- выделены приватные методы:
  - `hasMissingComponent(...)`
  - `isSelfLoop(...)`
  - `hasBlankSignalName(...)`

Дополнительно устранена преднамеренно внесенная ранее логическая ошибка в проверке отсутствующего source/target.

## 4. Обновление тестов

Добавлен новый тест:

- `findInvalidConnectionIds should return connection with blank signal name`

Существующие тесты на старые требования оставлены без изменений, чтобы подтвердить обратную совместимость поведения.

## 5. Регрессионное тестирование

Для проверки использован полный набор тестов проекта:

```bash
./gradlew test
```

Regression suite должен подтвердить:

- работоспособность новой логики;
- отсутствие поломок ранее реализованных модулей;
- корректную работу unit, integration и functional-support тестов.

## 6. Результат

В рамках лабораторной работы №11 выполнено изменение требований к функции, проведен рефакторинг кода и успешно пройден полный regression suite.
