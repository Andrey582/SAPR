# Лабораторная работа №5
## Разработка заглушек и black-box проверка невалидных данных

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 05.04.2026

## 1. Цель работы

Изолировать модули бизнес-логики с помощью `Mocks/Stubs` и проверить методы из лабораторной работы №4 при подаче невалидных данных методом черного ящика.

## 2. Разработанные заглушки

Для изоляции модулей введены интерфейсы-политики:

- `ComponentLabelPolicy` — управляет допустимостью имени компонента;
- `ConnectionPolicy` — управляет допустимостью соединения между компонентами.

В модульных тестах использованы:

- `stub` для возврата заранее заданного результата;
- `mock`-подобная реализация для фиксации переданного аргумента и проверки факта вызова.

## 3. Black-box проверка невалидных данных

Проверены невалидные входные данные для методов из ЛР4:

- пустое имя компонента;
- слишком длинное имя компонента;
- пустой идентификатор соединения;
- попытка соединить компонент с самим собой;
- пустое имя сигнала.

## 4. Реализованные тесты

Добавлены тестовые классы:

- `CircuitBlackBoxValidationTest`
- `CircuitServiceIsolationTest`

Они проверяют:

- обработку ошибочных входов без знания внутренней реализации;
- использование `stub/mock` для подмены зависимостей сервисов;
- корректную реакцию сервисов при запрете операции внешней политикой.

## 5. Результат

В рамках лабораторной работы №5 разработаны заглушки для изоляции модулей и реализованы black-box тесты, проверяющие методы из ЛР4 на невалидных данных.
