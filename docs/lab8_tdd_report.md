# Лабораторная работа №8
## Применение TDD при разработке новой функции

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 06.04.2026

## 1. Цель работы

Применить методологию `TDD (Test-Driven Development)`: сначала написать падающие тесты для нереализованной функции, затем реализовать код до получения зеленого прогона.

## 2. Выбранная функция

Для модуля анализа схемы добавлена новая функция:

- `findInvalidConnectionIds(components, connections)`

Назначение функции:

- проверять корректность соединений относительно списка компонентов схемы;
- возвращать идентификаторы некорректных соединений.

Соединение считается некорректным, если:

- отсутствует исходный компонент;
- отсутствует целевой компонент;
- соединение образует self-loop.

## 3. Шаги TDD

### Шаг 1. Red

Сначала были написаны 3 теста для еще не реализованной функции:

- корректное соединение не должно попадать в список ошибок;
- соединение с отсутствующим source должно считаться некорректным;
- соединение с self-loop должно считаться некорректным.

После запуска `./gradlew test` получен ожидаемый красный результат: компиляция тестов завершилась ошибкой `Unresolved reference`, так как функции еще не существовало.

### Шаг 2. Green

После этого в `CircuitAnalysisService` реализован метод `findInvalidConnectionIds`, который:

- собирает множество допустимых `component id`;
- фильтрует соединения с отсутствующими ссылками;
- фильтрует self-loop соединения;
- возвращает список идентификаторов некорректных соединений.

### Шаг 3. Refactor

Реализация оставлена компактной и встроена в существующий сервис анализа схемы без изменения публичного контракта других модулей.

## 4. Результат

В рамках лабораторной работы №8 методология TDD применена полностью: сначала созданы 3 падающих теста, затем реализован код новой функции, после чего тесты переведены в зеленое состояние.
