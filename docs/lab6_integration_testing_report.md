# Лабораторная работа №6
## Реализация взаимодействия модулей и интеграционное тестирование

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 06.04.2026

## 1. Цель работы

Реализовать взаимодействие двух модулей системы и провести интеграционное тестирование выбранным методом.

## 2. Взаимодействующие модули

В проекте интегрированы два основных модуля:

- модуль работы с компонентами схемы (`CircuitComponentService`);
- модуль работы с соединениями схемы (`CircuitConnectionService`).

Для фиксации результата взаимодействия дополнительно использован модуль анализа схемы (`CircuitAnalysisService`).

## 3. Реализация взаимодействия

Создан сервис `CircuitAssemblyService`, который:

- переименовывает исходный компонент;
- создает соединение между двумя компонентами;
- формирует итоговый набор компонентов и соединений;
- определяет изолированные компоненты схемы.

Результат сборки возвращается через модель `CircuitAssemblyResult`.

## 4. Метод интеграционного тестирования

Выбран **восходящий метод интеграционного тестирования**.

Причина выбора:

- в проекте уже существуют независимые низкоуровневые сервисы;
- архитектура позволяет сначала проверить отдельные сервисы, а затем их совместную работу через верхнеуровневый orchestration-сервис;
- для интеграции не требуется поднимать внешний API или UI-слой.

## 5. Интеграционные тесты

Добавлен класс `CircuitAssemblyIntegrationTest`, который проверяет:

- успешное взаимодействие модулей компонентов, соединений и анализа;
- корректное определение изолированного компонента при частичной сборке схемы;
- корректную реакцию интеграционного слоя на невалидные входные данные из нижележащих модулей.

## 6. Результат

В рамках лабораторной работы №6 реализовано взаимодействие модулей системы и выполнено интеграционное тестирование восходящим методом.
