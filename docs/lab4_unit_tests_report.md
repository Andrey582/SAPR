# Лабораторная работа №4
## Разработка базовой бизнес-логики и модульных тестов

**Дисциплина:** Система автоматизированного проектирования (САПР) схем  
**Дата выполнения:** 05.04.2026

## 1. Цель работы

Реализовать базовые классы бизнес-логики для проекта и покрыть их модульными тестами с использованием JUnit.

## 2. Реализованные классы бизнес-логики

- `CircuitComponentService` — перемещение компонента и изменение его подписи;
- `CircuitConnectionService` — создание корректного соединения между двумя разными компонентами;
- `CircuitAnalysisService` — анализ схемы: подсчет компонентов по типу и поиск изолированных элементов.

## 3. Подключение unit-фреймворка

В проекте используется `JUnit 5`. Для модульного тестирования задействованы зависимости из `build.gradle.kts`:

- `spring-boot-starter-test`
- `junit-jupiter`
- `kotlin-test-junit5`

Тестовая платформа запускается через `useJUnitPlatform()`.

## 4. Реализованные модульные тесты

Написано 7 модульных тестов:

- `CircuitComponentServiceTest`
- `CircuitConnectionServiceTest`
- `CircuitAnalysisServiceTest`

Проверяются сценарии:

- корректное изменение координат компонента;
- корректное переименование компонента;
- ошибка при пустом имени компонента;
- создание соединения между разными компонентами;
- ошибка при попытке соединить компонент с самим собой;
- подсчет компонентов по типу без учета регистра;
- поиск изолированных компонентов схемы.

## 5. Результат

В рамках лабораторной работы №4 реализованы 3 базовых класса бизнес-логики и 7 модульных тестов на JUnit 5, покрывающих основные методы системы.
